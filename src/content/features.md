# TeGo-AI 产品功能介绍

<!-- first-desc -->
TeGo-AI 智能体管理平台是一个全面的 AI 基础设施解决方案，通过六大核心功能模块，为企业提供高效、敏捷的 AI 能力，助力快速构建智能应用。平台采用微服务架构设计，支持容器化部署，提供完整的智能体生命周期管理、MCP 服务集成、动态界面生成等核心功能。

## 首页 {#home}

> 首页是用户进入系统的第一个界面，提供系统整体运行状态的概览，包括关键指标统计、快捷操作入口和实时数据展示。
<!-- label:功能概述 -->

**数据统计展示**
- **智能体总数统计**：实时显示平台内所有智能体的总数量，包括运行中、已停止、异常状态的智能体分布
- **系统资源使用情况**：展示 CPU、内存、磁盘、网络等系统资源的实时使用率和历史趋势
- **服务状态监控**：显示各个微服务的运行状态，包括健康检查结果和响应时间
- **用户活动统计**：记录用户登录次数、操作频率、最近活动等数据

**快捷操作入口**
- **智能体创建**：一键创建新的智能体实例，支持模板选择和快速配置
- **智能体管理**：直接进入智能体列表管理页面，查看所有智能体状态
- **MCP 服务**：快速访问 MCP 服务管理功能，查看服务状态和配置
- **系统设置**：进入系统配置页面，进行参数调整和权限管理

**实时监控面板**
- **性能指标图表**：展示系统关键性能指标的历史趋势和实时数据
- **告警信息展示**：显示当前活跃的告警信息，包括级别、描述和处理状态
- **最近活动记录**：展示用户最近的操作记录和系统事件日志
- **资源使用趋势**：通过图表形式展示系统资源使用的历史趋势和预测

## 智能体 {#agent}

> 智能体引擎是平台的核心组件，负责智能体的创建、配置、运行和监控，提供完整的智能体生命周期管理。智能体市场提供预置的智能体模板和配置，用户可以快速选择和使用，也可以分享自己创建的智能体模板。智能体管理模块提供对已创建智能体的全面管理功能，包括配置管理、状态监控、性能优化等。
<!-- label:功能概述 -->

### 智能体引擎 {#agent-engine}

**智能体创建与配置**
- **基础信息设置**：配置智能体名称、描述、标签、版本等基础信息
- **模型配置管理**：选择 LLM 模型类型（如 GPT-4、Claude、本地模型），设置模型参数（温度、最大长度、停止词等）
- **工具配置绑定**：为智能体绑定 MCP 服务工具，支持多个工具的配置和优先级设置
- **权限和安全设置**：配置智能体的访问权限、API 密钥、IP 白名单等安全参数

**智能体运行监控**
- **运行状态监控**：实时监控智能体的运行状态，包括启动、运行中、停止、异常等状态
- **性能指标统计**：收集智能体的响应时间、吞吐量、错误率、资源使用等性能指标
- **日志实时查看**：提供智能体运行日志的实时查看功能，支持日志级别过滤和关键词搜索
- **资源使用统计**：监控智能体的 CPU、内存、网络等资源使用情况

**智能体管理操作**
- **批量操作功能**：支持批量启动、停止、删除智能体，提高管理效率
- **智能体搜索和筛选**：根据名称、状态、标签、创建时间等条件搜索和筛选智能体
- **智能体克隆和备份**：支持智能体配置的克隆和备份，便于快速部署和恢复
- **版本管理**：管理智能体的不同版本，支持版本回滚和对比功能

### 智能体市场 {#agent-market}

**模板库管理**
- **预置模板展示**：提供多种类型的智能体模板，包括客服助手、数据分析师、代码助手等
- **模板分类浏览**：按功能类型、行业领域、复杂度等维度对模板进行分类
- **模板详情查看**：展示模板的详细配置信息、使用说明、性能指标等
- **模板评分和评论**：用户可以对模板进行评分和评论，帮助其他用户选择

**智能体部署**
- **一键部署功能**：选择模板后可以一键部署到自己的环境中
- **参数自定义**：在部署过程中可以自定义智能体的参数和配置
- **环境适配**：自动适配目标环境的配置要求，确保部署成功
- **部署状态跟踪**：实时跟踪部署进度，显示部署日志和结果

**模板分享机制**
- **模板发布**：用户可以将自己创建的智能体配置发布为模板
- **权限控制**：支持模板的公开、私有、指定用户可见等权限设置
- **版本管理**：管理模板的不同版本，支持版本更新和回滚
- **使用统计**：统计模板的使用次数、评分、下载量等数据

### 智能体管理 {#agent-management}

**智能体列表管理**
- **智能体列表展示**：以列表形式展示所有智能体，包括名称、状态、创建时间、最后修改时间等信息
- **状态管理**：支持智能体的启动、停止、重启、删除等状态操作
- **批量操作**：支持批量选择智能体进行批量操作，提高管理效率
- **搜索和筛选**：支持按名称、状态、标签、创建时间等条件搜索和筛选智能体

**智能体配置管理**
- **配置信息查看**：查看智能体的详细配置信息，包括模型配置、工具配置、权限设置等
- **配置修改**：支持修改智能体的各项配置参数，修改后需要重启生效
- **配置对比**：支持不同版本配置的对比，查看配置变更历史
- **配置导入导出**：支持智能体配置的导入导出，便于配置备份和迁移

**智能体监控分析**
- **实时监控**：实时监控智能体的运行状态、性能指标、资源使用情况
- **历史数据分析**：分析智能体的历史运行数据，包括响应时间、错误率、使用频率等
- **性能优化建议**：基于监控数据提供性能优化建议，帮助提升智能体性能
- **告警设置**：设置智能体的告警规则，当出现异常时及时通知

## MCP {#mcp}


> MCP 市场是 Model Context Protocol 服务的集中管理平台，提供丰富的 MCP 服务模板和工具，支持智能体与外部系统的集成。MCP 令牌管理模块提供对 MCP 服务访问令牌的创建、管理和监控功能，确保服务访问的安全性和可控性。使用日志模块提供对 MCP 服务的全面管理功能，包括服务配置、状态监控、性能优化等。
<!-- label:功能概述 -->


### MCP市场 {#mcp-market}

**服务模板库**
- **预置服务模板**：提供数据库查询、文件操作、API 调用、邮件发送等常用服务模板
- **服务分类管理**：按功能类型（工具类、数据类、API 类、通信类）对服务进行分类
- **服务详情展示**：展示服务的功能描述、参数说明、使用示例、性能指标等
- **服务评分系统**：用户可以对服务进行评分和评论，帮助选择优质服务

**服务浏览和搜索**
- **分类浏览**：按功能类型、行业领域、使用频率等维度浏览服务
- **关键词搜索**：支持服务名称、描述、标签等关键词搜索
- **筛选功能**：根据服务类型、评分、使用次数、更新时间等条件筛选
- **推荐服务**：基于用户使用历史和偏好推荐相关服务

**服务试用和测试**
- **在线试用功能**：提供服务的在线试用环境，用户可以测试服务功能
- **参数配置测试**：支持服务参数的在线配置和测试
- **性能测试工具**：提供服务的性能测试工具，评估响应时间和稳定性
- **测试报告生成**：生成详细的测试报告，包括功能验证和性能评估

### MCP令牌 {#mcp-token}

**令牌创建和管理**
- **令牌生成**：为 MCP 服务生成访问令牌，支持不同类型的令牌（API 令牌、用户令牌、服务令牌等）
- **令牌配置**：配置令牌的权限范围、有效期、使用限制等参数
- **令牌更新**：支持令牌的更新和续期，确保服务的持续访问
- **令牌撤销**：支持令牌的撤销和禁用，及时停止无效访问

**令牌权限控制**
- **权限范围设置**：设置令牌可以访问的服务和功能范围
- **访问限制配置**：配置令牌的访问频率限制、IP 白名单等安全限制
- **权限继承机制**：支持基于用户角色和权限的令牌权限继承
- **动态权限调整**：支持令牌权限的动态调整，无需重新生成令牌

**令牌使用监控**
- **使用情况统计**：统计令牌的使用次数、访问频率、成功率等数据
- **异常访问检测**：检测异常的访问行为，如高频访问、异常 IP 等
- **访问日志记录**：记录令牌的详细访问日志，支持审计和追踪
- **性能影响分析**：分析令牌使用对系统性能的影响，优化访问策略

### 使用日志 {#mcp-logs}

**日志记录功能**
- **访问日志记录**：记录所有 MCP 服务的访问日志，包括访问时间、用户、服务、参数等
- **操作日志记录**：记录用户对 MCP 服务的操作日志，包括创建、修改、删除等操作
- **错误日志记录**：记录服务访问过程中的错误信息，便于问题排查
- **性能日志记录**：记录服务的响应时间、吞吐量等性能指标

**日志查询和分析**
- **多维度查询**：支持按时间、用户、服务、状态等多维度查询日志
- **关键词搜索**：支持日志内容的关键词搜索，快速定位相关信息
- **日志过滤**：支持按日志级别、服务类型、用户等条件过滤日志
- **统计分析**：提供日志数据的统计分析功能，生成使用报告

**日志管理功能**
- **日志存储管理**：管理日志的存储策略，包括存储周期、压缩策略等
- **日志备份**：支持日志数据的备份和恢复，确保数据安全
- **日志清理**：支持过期日志的自动清理，优化存储空间
- **日志导出**：支持日志数据的导出，便于离线分析和审计

### MCP管理 {#mcp-management}

**服务配置管理**
- **服务参数设置**：配置服务的连接参数、功能参数、性能参数、安全参数等
- **服务依赖管理**：配置服务之间的依赖关系，确保启动顺序正确
- **服务发现机制**：自动发现和注册服务，支持动态服务发现
- **负载均衡配置**：配置服务的负载均衡策略，提高系统可用性

**服务状态监控**
- **运行状态检查**：实时监控服务的运行状态，包括启动、运行、停止、异常等
- **健康检查机制**：定期进行服务健康检查，确保服务可用性
- **性能指标监控**：监控服务的响应时间、吞吐量、错误率等性能指标
- **资源使用监控**：监控服务占用的 CPU、内存、网络等资源

**服务运维管理**
- **服务部署**：支持服务的自动化部署和更新
- **服务扩缩容**：支持服务的自动扩缩容，根据负载情况调整实例数量
- **故障转移设置**：配置服务的故障转移机制，确保服务高可用
- **服务版本管理**：管理服务的不同版本，支持版本回滚和升级

## 模型网关 {#model-gateway}

> 模型网关管理模块提供对 LLM 模型的统一接入和管理功能，支持多种模型类型的集成和路由。
<!-- label:功能概述 -->
### 网关管理 {#gateway-management}

**模型接入管理**
- **模型注册**：注册各种类型的 LLM 模型，包括 OpenAI、Claude、本地模型等
- **模型配置**：配置模型的连接参数、认证信息、API 版本等
- **模型测试**：提供模型连接测试功能，确保模型可以正常访问
- **模型状态监控**：监控模型的可用性和响应状态

**路由策略管理**
- **路由规则配置**：配置模型的路由规则，根据请求类型、负载情况等选择合适模型
- **负载均衡策略**：配置模型的负载均衡策略，包括轮询、权重、最少连接等
- **故障转移机制**：配置模型的故障转移机制，当主模型不可用时自动切换到备用模型
- **路由性能优化**：优化路由策略，提高模型访问的响应速度和成功率

**访问控制管理**
- **访问权限控制**：控制用户对不同模型的访问权限
- **使用配额管理**：设置用户对模型的使用配额和限制
- **访问频率限制**：配置模型访问的频率限制，防止滥用
- **访问日志记录**：记录模型访问的详细日志，支持审计和分析

## 系统管理 {#system-management}

> 系统设置模块提供系统的基础配置、安全设置和备份恢复功能，确保系统的安全性和可靠性。用户管理模块提供完整的用户账户管理功能，包括用户创建、信息维护、状态控制等。角色权限模块提供完整的角色定义和权限管理功能，确保系统的安全性和可控性。
<!-- label:功能概述 -->
### 系统设置 {#system-settings}

**基础配置**
- **系统参数设置**：配置系统的基础参数，包括端口、超时时间、缓存大小等
- **环境变量配置**：管理系统的环境变量，支持不同环境的配置切换
- **日志级别设置**：配置系统的日志级别，平衡日志详细度和性能
- **缓存策略配置**：配置系统的缓存策略，提高系统性能

**安全设置**
- **密码策略配置**：配置用户密码的复杂度要求和有效期
- **会话管理设置**：配置用户会话的超时时间和并发限制
- **防火墙规则**：配置系统防火墙规则，控制网络访问
- **数据加密配置**：配置数据传输和存储的加密方式

**备份恢复**
- **数据备份策略**：配置自动备份策略，包括备份频率、保留时间等
- **备份计划管理**：管理备份计划，支持手动和自动备份
- **数据恢复功能**：提供数据恢复功能，支持全量和增量恢复
- **备份历史记录**：记录备份历史，支持备份验证和清理

### 用户管理 {#user-management}

**用户账户管理**
- **用户创建**：创建新的用户账户，设置基本信息、联系方式、部门等
- **用户信息维护**：维护用户的基本信息、联系方式、部门归属等数据
- **用户状态控制**：控制用户的启用、禁用、锁定等状态
- **用户删除**：支持用户账户的删除和恢复操作

**用户组织管理**
- **部门管理**：管理公司的部门结构，支持多级部门设置
- **用户分组**：将用户按功能、项目等维度进行分组管理
- **组织架构维护**：维护公司的组织架构，包括部门关系、汇报关系等
- **用户调动管理**：管理用户的部门调动和角色变更

**用户操作审计**
- **操作日志记录**：记录用户的所有操作日志，包括登录、功能访问、数据操作等
- **登录历史查看**：查看用户的登录历史，包括登录时间、IP 地址、设备信息等
- **异常行为检测**：检测用户的异常行为，如异常登录、频繁操作等
- **审计报告生成**：生成用户操作的审计报告，支持合规性检查

### 角色权限 {#role-permissions}

**角色管理**
- **角色定义**：定义系统中的各种角色，如管理员、普通用户、访客等
- **角色继承关系**：配置角色之间的继承关系，简化权限管理
- **角色模板管理**：提供预置的角色模板，快速创建常用角色
- **角色权限继承**：支持角色权限的继承和覆盖机制

**权限配置**
- **功能权限设置**：配置用户对系统功能的访问权限，如菜单访问、按钮操作等
- **数据权限控制**：控制用户对数据的访问范围和操作权限，如数据查看、编辑、删除等
- **API 访问权限**：配置用户对 API 接口的访问权限，确保接口安全
- **资源使用限制**：设置用户对系统资源的使用限制，如存储空间、API 调用次数等

**权限审计**
- **权限变更记录**：记录权限的变更历史，包括变更时间、变更人、变更内容等
- **权限使用统计**：统计权限的使用情况，包括使用频率、使用效果等
- **权限合规检查**：检查权限配置的合规性，确保符合安全要求
- **权限优化建议**：基于使用情况提供权限优化建议，提高权限管理效率
